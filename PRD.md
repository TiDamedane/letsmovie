# 去看电影吧 / FrameClub MVP PRD

版本：0.1.0  
更新日期：2026-06-16  
产品形态：移动端优先的本地 Web App  
目标画布：393 × 852  
当前实现：React + Tailwind + shadcn/ui，本地 LocalStorage 持久化

## 1. 产品定位

「去看电影吧」是一款帮助朋友一起发起观影活动、收集候选影片、完成轻量挑选，并在观影后留下回忆的生活方式产品。

它不是后台管理工具，也不是标准投票工具。产品气质应接近深夜电影沙龙、DVD 收藏架、黑胶唱片店和朋友之间的安静仪式。

核心原则：

- Friends First：先呈现一起看电影的人，再呈现电影。
- Movie Poster is the Hero：海报始终是电影相关页面的视觉主角。
- Quiet Ritual：关键状态变化需要有仪式感，但不过度喧闹。
- Recommendation Should Not Interrupt Movies：推荐影片应该轻量、顺手，不打断浏览。
- Remember Who You Watched It With：最终留下的不只是电影结果，而是这次和朋友一起看的记忆。

## 2. MVP 目标

MVP 需要完成一条完整闭环：

1. 用户创建观影活动。
2. 活动出现在「我的活动」。
3. 用户进入活动详情。
4. 用户推荐影片，影片进入候选影片池。
5. 主持人开始挑选。
6. 用户必须挑选 3 部候选影片。
7. 系统进行随机抽取并揭晓今晚影片。
8. 活动状态变为「影片已选定」。
9. 用户留下今晚回忆。
10. 系统生成伪票根。
11. 活动从「我的活动」归档到「我的回忆」。
12. 用户可在「我的回忆」中查看或移除这张回忆票根。

## 3. 用户与角色

### 3.1 当前用户

- 默认用户：小杨。
- 默认身份：活动主持人。
- 当前 MVP 不包含登录、注册、账号切换和真实多人协作。

### 3.2 主持人

主持人是 Event Host，不是管理员。

当前主持人能力：

- 创建活动。
- 推荐影片。
- 开始挑选。
- 解散活动。
- 留下回忆。
- 移除已归档回忆。

### 3.3 成员

当前版本使用固定头像模拟成员。

成员在 MVP 中主要用于：

- 活动卡片与活动详情的社交氛围展示。
- 回忆票根详情中的成员头像选择区。
- 后续多人回忆数据扩展的占位。

## 4. 信息架构与路由

### 4.1 我的活动

路由：`#/`

职责：

- 展示尚未归档的活动。
- 提供底部导航。
- 提供创建活动入口。
- 点击活动卡进入活动详情。

页面状态：

- 默认情况下可以为空。
- 活动归档后从本页面移除。
- 从活动详情返回本页时，活动详情页使用票根同款退出动画。

### 4.2 创建观影局

路由：`#/activities/new`

职责：

- 填写活动名称、留言、地点、日期。
- 创建活动并返回「我的活动」。

当前日期粒度：

- 只需要年、月、日。
- 不需要精确到小时和分钟。

### 4.3 活动详情

路由：`#/activities/:activityId`

职责：

- 展示活动 Hero、状态、主持人、成员、候选影片池。
- 推荐影片。
- 开始挑选。
- 完成 3 部影片选择。
- 随机抽取今晚影片。
- 留下回忆。
- 解散活动。

### 4.4 我的回忆

路由：`#/memories`

职责：

- 展示已经归档的活动卡。
- 归档卡按活动开始日期从近到远排序。
- 如果活动开始日期相同，则按生成归档票根的时间排序。
- 点击卡片进入回忆票根详情。

### 4.5 回忆票根详情

路由：`#/memories/:activityId`

职责：

- 展示该活动归档后的伪票根。
- 支持查看成员头像与成员留下的回忆气泡。
- 支持返回「我的回忆」。
- 支持通过右上角更多菜单移除回忆。

## 5. 底部导航

底部导航用于首页级页面，包括：

- 我的活动
- 中央创建按钮
- 我的回忆

规则：

- 位于手机画布底部固定位置。
- 背景为深色圆角胶囊。
- 中间按钮使用品牌纯色。
- 左右文字按钮字号为 14px。
- 文字使用奶油白。
- 当前选中项不透明度为 100%。
- 未选中项不透明度为 60%。

## 6. 核心状态

### 6.1 影片收集中 `collecting`

默认状态。

用户可以：

- 查看活动信息。
- 查看主持人与成员。
- 打开推荐影片面板。
- 搜索影片。
- 加入或移除候选影片。
- 点击「开始挑选」进入确认弹窗。
- 解散活动。

限制：

- 电影卡不可被挑选。
- 候选影片池可为空。

### 6.2 影片挑选中 `picking`

进入条件：

- 用户点击「开始挑选」。
- 确认弹窗中点击「确认开始」。
- 二次反馈中点击「知道了」。

用户可以：

- 点击候选影片卡。
- 最多选择 3 部。
- 必须选择满 3 部后才能确认。

视觉：

- 被选中的电影卡上浮。
- 出现轻微旋转。
- 出现品牌色边框。
- 出现酒红色低透明覆盖层。
- 右上角显示心形状态。

### 6.3 影片已选定 `selected`

进入条件：

- 用户完成 3 部选择。
- 点击底部确认按钮。
- 系统通过随机抽取模式揭晓最终影片。
- 用户点击「确认今晚影片」。

状态变化：

- Hero Banner 背景切换为最终影片海报。
- 状态文案切换为「影片已选定」。
- 状态圆点改为绿色。
- 被选中的最终电影卡保持选中状态。
- 其他候选影片卡降低透明度，用于突出最终影片。
- 底部按钮切换为「留下回忆」。

## 7. 功能需求

### 7.1 创建活动

字段：

- 活动名称：必填，最多 30 字，默认空。
- 留言：选填，最多 100 字，默认空。
- 地点：必填，默认空。
- 日期：默认 `2026.07.20`，支持选择年、月、日。

创建规则：

- 活动名称或地点为空时不可创建。
- 创建成功后写入 LocalStorage。
- 新活动状态为 `collecting`。
- 候选影片池为空。
- 默认当前用户与固定成员已加入。
- 创建成功后返回「我的活动」。

动画：

- 创建页从底部进入。
- 返回时使用反向退出动画。
- 日期选择器从底部进入。
- 点击确认日期后使用反向退出动画。

### 7.2 推荐影片

入口：

- 活动详情底部「推荐影片」。

面板：

- 从手机画布底部弹出。
- 颜色为 Elevated Card。
- 点击面板外区域时，面板退回，而不是瞬间消失。
- 面板内部可滚动，不与外部页面滚动冲突。

搜索：

- 支持按片名搜索。
- 支持按导演搜索。
- 输入变化后，如果不再匹配，旧结果消失。
- 清除按钮使用品牌色。

当前影片库：

| 影片 | 导演 |
| --- | --- |
| 花样年华 | 王家卫 |
| 重庆森林 | 王家卫 |
| 楚门的世界 | 彼得·威尔 |
| 罗拉快跑 | 汤姆·提克 |
| 被嫌弃的松子的一生 | 中岛哲也 |
| 闻香识女人 | 马丁·布莱斯 |
| 四百击 | 弗朗索瓦·特吕弗 |

加入规则：

- 点击加号加入候选影片池。
- 加入后该搜索结果出现低透明酒红色选中层。
- 加号变为勾号。
- 再次点击可移除。
- 候选池中新增电影卡有轻微上移动画。
- 由当前用户推荐的影片，推荐人显示为「小杨」。

### 7.3 候选影片池

布局：

- 三列布局。
- 海报比例 2:3。
- 电影卡圆角 16px。
- 片名与推荐人覆盖在海报底部。
- 默认整齐排列。

空状态：

- 文案：「现在还没有人推荐影片」。
- 文案位于候选影片卡片内容区中间。

### 7.4 开始挑选

点击「开始挑选」后出现确认弹窗。

确认弹窗：

- 卡片颜色与 Elevated Card 一致。
- 包含打板图标。
- 文案：「大家推荐得差不多了吗？」
- 辅助文案：「开始后，候选影单将暂时固定下来」
- 操作：
  - 再看看
  - 确认开始

交互：

- 点击「再看看」或弹窗外区域时，弹窗使用反向动画退出。
- 点击「确认开始」后，弹窗文案切换为：
  - 「现在可以邀请朋友们开始挑选了」
- 图标变为打板合上状态。
- 按钮只保留「知道了」。
- 点击「知道了」后状态进入 `picking`。

### 7.5 挑选影片

规则：

- 仅 `picking` 状态可点击电影卡。
- 最多选择 3 部。
- 必须选择 3 部才能继续。
- 底部按钮实时显示 `N / 3`。
- 未满 3 部时按钮不可点击。
- 满 3 部时按钮可点击。

### 7.6 随机抽取模式

MVP 使用轻便的随机抽取模式，不做完整投票统计。

触发：

- 用户挑选满 3 部后点击确认按钮。

流程：

1. 页面保持暗色遮罩。
2. 屏幕中央出现圆角海报容器。
3. 容器内海报以老虎机形式上下滚动。
4. 前半程快速滚动。
5. 后半程逐渐减速。
6. 总时长约 3 秒。
7. 最终停留在随机选中的影片海报。
8. 随后依次浮现：
   - 揭晓影片
   - 电影名
   - 推荐人
   - 确认今晚影片按钮

视觉规则：

- 不使用闪烁式切图。
- 上一张海报向上离场。
- 下一张海报从下方进入。
- 动效使用 ease-out 或 cubic-bezier。
- 不使用弹跳。

### 7.7 确认今晚影片

点击「确认今晚影片」后：

- 活动状态写入 `selected`。
- 写入 `selectedMovieId`。
- Hero Banner 切换为最终影片海报。
- 状态变为「影片已选定」。
- 状态点为绿色。
- 底部操作变为「留下回忆」。

### 7.8 留下回忆

入口：

- 活动状态为 `selected` 后，底部显示「留下回忆」。

回忆卡片：

- 从底部向上弹出。
- 覆盖整个手机画布。
- 背景颜色与创建观影局卡片一致。
- 左上角有退出按钮。
- 点击退出或卡片外区域后，回忆卡片关闭，并清空本次未提交内容。
- 下次进入重新走选择流程。

内容：

- 标题：「今晚怎么样？」
- 7 个 Emoji：
  - 😭
  - 😂
  - 😅
  - 😡
  - 😌
  - 😍
  - 😴

交互：

- 选中任一 Emoji 后，该 Emoji 出现品牌色外边框。
- 选中后，下方内容从下往上浮现：
  - 「发生什么了？」
  - 留言输入框
  - 「记住今晚」按钮
- 留言不是必填。

提交：

- 点击「记住今晚」后生成伪票根。
- 同一活动只在第一次生成票根时分配票根编号。
- 票根编号根据用户累计完成「留下回忆」的次数递增：
  - 第一次为 `NO.1`
  - 第二次为 `NO.2`
  - 以此类推

### 7.9 即时伪票根

出现时机：

- 用户点击「记住今晚」后立即出现。

行为：

- 票根浮现动画与回忆详情票根一致。
- 点击票根任意区域后：
  - 票根退出。
  - 活动写入 `archivedAt`。
  - 活动从「我的活动」中上浮并消失。
  - 活动进入「我的回忆」。

票根内容：

- 上半部分为最终电影海报。
- 右上角显示 `NO.X`。
- 海报内显示活动名称。
- 下半部分显示：
  - 电影名称
  - 地点
  - 日期，格式为 `YYYY.M.D`
  - 用户选择的 Emoji
  - 用户填写的留言

字体：

- 活动标题和电影名称使用 `Source Han Serif CN Medium`。
- `NO.X` 使用与活动标题相同的字体。

### 7.10 我的回忆

入口：

- 底部导航「我的回忆」。

展示规则：

- 仅展示已归档活动。
- 按活动开始日期从近到远排序。
- 日期相同时，按生成归档票根的时间排序。

卡片：

- 复用活动卡片视觉结构。
- 点击后进入回忆票根详情。

### 7.11 回忆票根详情

路由：`#/memories/:activityId`

区别于即时伪票根：

- 不允许点击任意区域关闭。
- 左上角有返回按钮。
- 右上角有更多按钮。
- 只有返回或移除时才退出。
- 退出动画复用即时票根退出动画。

内容：

- 上半部分为最终电影海报。
- 右上角显示 `NO.X`。
- 海报内显示活动名称。
- 下半部分显示：
  - 电影名称
  - 地点
  - 日期
  - 成员头像列表

成员头像：

- 头像横向排列并均匀分布。
- 超出一行时自动换行。
- 点击某个头像后，该头像出现品牌色外边框。
- 如果该成员有回忆内容，下方从下往上浮现回忆气泡。

回忆气泡：

- 气泡需要有尖角。
- 气泡内左侧显示 Emoji 圆形。
- 右侧显示该成员留下的文字。
- Emoji 与文字第一行应垂直居中。
- 留言为空时显示空文案。

更多菜单：

- 点击右上角更多按钮后，从底部弹出操作卡片。
- 当前仅包含「移除回忆」。
- 不显示前置图标。
- 点击外部区域时，操作卡片使用与活动操作面板一致的回退动画。
- 点击「移除回忆」后删除该归档记录，并返回「我的回忆」。

### 7.12 解散活动

入口：

- 活动详情右上角更多按钮。

行为：

- 从底部弹出活动操作卡片。
- 当前仅包含「解散活动」。
- 不显示前置图标。
- 点击外部区域时，操作卡片以反向动画关闭。
- 点击「解散活动」后删除该活动并返回「我的活动」。

### 9.5 动效

原则：

- Slow，像电影，而不是短视频。
- 重要状态有仪式感。
- 不使用弹跳。
- 退出动画应与进入动画方向相反。

关键动效：

- Sheet：从底部进入，反向退出。
- Modal：轻微浮现，反向退出。
- 活动详情返回：首页方向退出，复用票根退出动画。
- 票根：浮现进入，返回或移除时反向退出。
- 随机揭晓：老虎机式上下滚动。
- 活动归档：首页卡片上浮并消失。

## 10. 非功能需求

- 移动端优先，核心体验按 393 × 852 验证。
- 桌面端仅居中展示手机画布，不扩展为后台式桌面布局。
- 核心数据离线可用。
- 图片资源使用本地静态资源。
- 表单、按钮、弹窗应具备语义化标签。
- 浏览器 LocalStorage 异常时应回退为空列表，不阻断页面渲染。
- 不引入登录、服务端和网络依赖。

## 11. MVP 验收标准

1. 清空 LocalStorage 后，「我的活动」为空。
2. 底部导航显示「我的活动 / 创建按钮 / 我的回忆」。
3. 未填写活动名称或地点时无法创建活动。
4. 创建活动后，活动出现在「我的活动」。
5. 新活动详情中的候选影片池为空，并显示「现在还没有人推荐影片」。
6. 推荐影片面板可通过片名或导演搜索影片。
7. 加入影片后，候选影片池、数量和本地数据同步更新。
8. 开始挑选后，状态持久化为 `picking`。
9. 挑选阶段最多选择 3 部影片。
10. 未选满 3 部时无法继续。
11. 选满 3 部后可触发随机抽取。
12. 随机抽取表现为上下滚动的老虎机动画，而不是闪烁切换。
13. 确认今晚影片后，状态变为 `selected`。
14. Hero Banner 切换为最终影片海报。
15. 最终影片卡保持选中，其他候选影片降低透明度。
16. 点击「留下回忆」后出现全屏回忆卡片。
17. 选中 Emoji 后才显示留言框和「记住今晚」。
18. 生成票根后，编号按用户累计完成次数递增。
19. 点击即时票根后，活动归档到「我的回忆」。
20. 归档时，活动卡从「我的活动」上浮并消失。
21. 「我的回忆」按日期和归档时间正确排序。
22. 点击回忆卡进入回忆票根详情。
23. 回忆票根不能通过任意点击关闭。
24. 点击成员头像后，头像出现品牌色边框，并显示对应回忆气泡。
25. 右上角更多菜单可打开「移除回忆」。
26. 点击菜单外区域时，操作卡片有回退动画。
27. 解散活动和移除回忆按钮都不显示前置图标。
28. 解散活动后，该活动从本地数据中删除。
29. 移除回忆后，该回忆从「我的回忆」中删除。
30. 所有页面、Sheet、Modal、票根和操作面板均有对应退出动画。
