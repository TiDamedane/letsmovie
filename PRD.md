# 去看电影吧（FrameClub）产品需求文档

版本：MVP 0.1  
更新日期：2026-06-14  
产品形态：移动端优先的 Web App  
目标画布：393 × 852

## 1. 产品概述

### 1.1 产品定位

「去看电影吧」是一款帮助朋友共同组织观影活动、推荐候选影片并完成观影选择、留下回忆的轻量社交产品。

它不是投票工具，不是排行榜，不是影评记录工具。应更接近深夜电影沙龙、海报收藏架和朋友间的共同仪式。

记录的不只是电影，更是一起看电影的人。

核心原则：

- Friends First：先呈现一起看电影的人，再呈现电影。
- Movie Poster is the Hero：海报始终是影片相关界面的视觉主角。
- Quiet Ritual：重要状态变化应有克制、缓慢且明确的仪式感。
- Recommendation Should Not Interrupt Movies：推荐操作应轻量，不打断用户浏览当前片单。

### 1.2 用户问题

- 朋友想一起看电影，但缺少一个集中收集候选影片的地方。
- 群聊中的推荐信息分散，难以确认当前有哪些候选影片。
- 直接发起投票过于工具化，缺少共同挑选电影的参与感。
- 活动时间、地点、成员和片单缺少统一承载页面。

### 1.3 产品目标

MVP 需要完成以下闭环：

1. 用户创建一个观影活动。
2. 活动出现在「我的活动」首页。
3. 用户进入活动详情并推荐影片。
4. 推荐影片进入候选影池。
5. 主持人结束收集并开启挑选。
6. 用户必须选择三部影片，才能确认挑选。
7. 主持人可以解散活动。

## 2. 用户与角色

### 2.1 当前用户

- 默认用户：小杨
- 默认身份：活动主持人
- 当前版本不包含登录、注册和账号切换。

### 2.2 主持人

主持人是 Event Host，不是管理员。

权限：

- 创建活动。
- 推荐或移除候选影片。
- 开启影片挑选。
- 解散活动。

### 2.3 成员

当前版本使用固定成员头像模拟活动成员。

成员能力暂不区分，邀请、加入和多人实时协作不在本期实现范围内。

## 3. 信息架构

### 3.1 我的活动

路由：`#/`

职责：

- 展示用户已创建的活动。
- 无活动时保持空状态。
- 通过底部主操作进入创建活动页面。
- 点击活动卡进入对应活动详情。

### 3.2 创建观影局

路由：`#/activities/new`

职责：

- 填写活动名称、留言、地点和时间。
- 创建活动并返回「我的活动」。

### 3.3 活动详情

路由：`#/activities/:activityId`

职责：

- 展示活动信息、主持人、成员及候选影池。
- 推荐影片。
- 从收集阶段进入挑选阶段。
- 完成三部影片的选择。
- 解散活动。

## 4. 核心状态

### 4.1 影片收集中 `collecting`

- 新活动的默认状态。
- 候选影池默认为空。
- 用户可以打开推荐影片面板。
- 用户可以搜索并加入或移除候选影片。
- 电影卡不可被挑选。
- 底部操作为「推荐影片」和「开始挑选」。

### 4.2 影片挑选中 `picking`

- 主持人确认开始挑选后进入。
- 候选影片停止增删。
- 电影卡变为可选择状态。
- 每位用户最多且必须选择三部影片。
- 底部显示选择进度和「确认挑选」。
- 未选满三部时按钮不可点击。
- 选满三部后按钮进入可点击状态。

### 4.3 后续状态

以下状态尚未在当前版本实现：

- 投票结果汇总。
- 最终影片揭晓。
- 观影完成与活动回忆。

## 5. 功能需求

### 5.1 我的活动

#### 默认状态

- 首次进入时不预置活动卡。
- 页面标题为「我的活动」。
- 底部显示固定操作区和创建按钮。

#### 活动列表

- 新创建的活动排在最前。
- 活动卡展示：
  - 当前状态。
  - 活动名称。
  - 地点。
  - 日期和时间。
  - 主持人头像。
  - 成员头像及超出人数。
- 点击整张卡片进入活动详情。

#### 创建入口

- 底部创建按钮使用品牌纯色。
- 点击后，创建页面从屏幕底部向上进入。

### 5.2 创建观影局

#### 活动名称

- 必填。
- 最多 30 个字符。
- 默认值为空。
- 空状态保留引导语。
- 实时显示字符计数。

#### 留言

- 选填。
- 最多 100 个字符。
- 默认值为空。
- 空状态保留引导语。
- 实时显示字符计数。

#### 地点

- 必填。
- 默认值为空。
- 使用手动输入框。

#### 时间

- 默认时间：2026.07.20 19:00。
- 点击时间区域后，从底部弹出滚轮选择器。
- 支持选择日期、0 至 23 时、0 至 59 分。
- 点击「确认时间」后保存草稿时间，并以反向动画收起面板。

#### 创建规则

- 活动名称或地点为空时，不允许创建。
- 创建成功后：
  - 生成唯一活动 ID。
  - 状态设为 `collecting`。
  - 候选影池设为空。
  - 写入本地存储。
  - 返回「我的活动」。

#### 返回

- 点击左上角返回时不保存未提交内容。
- 页面以进入动画的反向方式退出。

### 5.3 活动详情

#### Hero

- 展示活动名称、时间、地点、状态和留言。
- 收集阶段使用酒红色渐变与颗粒感背景。
- 当前状态使用圆点加状态文字表达。
- 状态文案：
  - `collecting`：影片收集中。
  - `picking`：影片挑选中。

#### 人员信息

- 展示主持人头像。
- 展示成员头像与 `+N`。
- 当前版本使用固定模拟成员。

#### 候选影池

- 三列等宽布局。
- 海报比例为 2:3。
- 电影卡圆角 16px。
- 片名和推荐人覆盖在海报底部。
- 空状态文案为「现在还没有人推荐影片」，并位于候选影池内容区域中央。

### 5.4 推荐影片

#### 打开方式

- 点击底部「推荐影片」后，从当前手机画布底部弹出面板。
- 面板不跳转新页面。
- 点击面板外区域时，以反向动画退回。
- 面板内部滚动与外部页面滚动互不冲突。

#### 搜索

- 搜索框支持电影名称和导演。
- 用户输入完整或部分关键词并按回车后执行搜索。
- 输入内容发生变化后，旧搜索结果不再保留。
- 无匹配结果时显示明确空状态。
- 清除按钮使用品牌色。

#### 搜索结果

- 当前电影库包含：

| 影片 | 导演 |
| --- | --- |
| 花样年华 | 王家卫 |
| 重庆森林 | 王家卫 |
| 楚门的世界 | 彼得·威尔 |
| 罗拉快跑 | 汤姆·提克 |
| 被嫌弃的松子的一生 | 中岛哲也 |
| 闻香识女人 | 马丁·布莱斯 |
| 四百击 | 弗朗索瓦·特吕弗 |

- 搜索结果展示海报、片名和导演。
- 海报尺寸与候选影池中的电影卡保持一致。

#### 加入候选影池

- 点击加号后：
  - 搜索结果卡出现低透明度酒红色选中层。
  - 加号切换为勾号。
  - 影片加入候选影池。
  - 新电影卡以 200ms 轻微上移并淡入。
- 再次点击勾号可从候选影池移除。
- 当前用户推荐的影片，推荐人统一记录为「小杨」。
- 候选影片 ID 持久化至当前活动。

### 5.5 开始挑选

#### 确认弹窗

- 点击「开始挑选」后展示居中确认弹窗。
- 弹窗包含打板图标、原始确认文案和两个操作：
  - 再看看。
  - 确认开始。
- 点击弹窗外区域或「再看看」时，使用进入动画的反向动画关闭。

#### 二次反馈

- 点击「确认开始」后：
  - 打板图标上部合拢。
  - 文案切换为「现在可以邀请朋友们开始挑选了」。
  - 操作只保留「知道了」。
- 点击「知道了」后：
  - 活动状态更新为 `picking`。
  - 状态文字更新为「影片挑选中」。
  - 弹窗反向退出。
  - 底部操作区切换为挑选进度按钮。

### 5.6 挑选影片

- 仅在 `picking` 状态下允许点击电影卡。
- 选中电影卡时：
  - 上浮约 7px。
  - 产生轻微随机旋转。
  - 显示酒红色边框。
  - 显示低透明度酒红色覆盖层。
  - 右上角显示心形状态图标。
- 再次点击可取消。
- 最多选择三部。
- 必须选择三部后才能确认。
- 底部按钮实时显示 `N / 3`。

当前版本仅实现选择及按钮可用状态，确认后的业务结果尚未定义。

### 5.7 解散活动

- 点击活动详情右上角更多按钮，从底部弹出活动操作面板。
- 面板当前仅包含「解散活动」。
- 点击后：
  - 面板以反向动画退出。
  - 删除本地活动数据。
  - 返回「我的活动」。

## 6. 数据模型

### 6.1 Activity

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | string | 本地生成的活动唯一 ID |
| title | string | 活动名称 |
| note | string | 活动留言 |
| location | string | 活动地点 |
| date | string | 日期，格式 `YYYY.MM.DD` |
| hour | string | 小时 |
| minute | string | 分钟 |
| status | collecting / picking | 活动状态 |
| candidateMovieIds | string[] | 候选影片 ID |
| createdAt | string | ISO 创建时间 |

### 6.2 Movie

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | string | 当前使用片名作为 ID |
| title | string | 片名 |
| director | string | 导演 |
| recommender | string | 推荐人 |
| src | string | 本地海报地址 |

### 6.3 本地存储

- 存储介质：浏览器 `LocalStorage`。
- Key：`letsmovie.activities.v1`。
- 当前版本不包含云端同步、冲突解决和数据迁移。

- Fast：200ms。

- 支持 `prefers-reduced-motion`。

## 8. 非功能需求

- 移动端优先，核心交互按 393 × 852 验证。
- 最小支持宽度 320px。
- 桌面端仅居中展示手机画布，不扩展为桌面后台布局。
- 使用语义化按钮、可访问名称和状态属性。
- 海报图片使用本地静态资源。
- 首屏和核心操作不依赖网络。
- 数据异常或 LocalStorage 解析失败时，回退为空活动列表。

## 9. MVP 验收标准

1. 清空 LocalStorage 后，「我的活动」不显示活动卡。
2. 未填写活动名称或地点时无法创建。
3. 创建成功后首页出现信息一致的活动卡。
4. 新活动候选影池为空。
5. 可通过片名或导演搜索到电影库中的影片。
6. 加入影片后，候选影池、数量和本地数据同步更新。
7. 移除影片后，候选影池和本地数据同步更新。
8. 开始挑选确认完成后，活动状态持久化为 `picking`。
9. 挑选阶段最多选择三部影片。
10. 未选满三部时无法确认，选满三部后按钮可用。
11. 解散活动后，该活动从首页和本地存储中消失。
12. 创建页、时间选择器、推荐面板、确认弹窗和活动操作面板均有对应退出动画。

## 10. 暂不纳入本期

- 用户注册与登录。
- 邀请码、分享链接和成员真实加入。
- 多设备或多人实时同步。
- 在线电影数据库和模糊联想搜索。
- 用户权限校验。
- 最终投票统计与开奖。
- 观影结果页、历史活动和回忆相册。
- 推送通知。
- 服务端数据库和数据备份。

## 11. 下一阶段建议

- 定义「确认挑选」后的结果和下一状态。
- 为挑选结果提供持久化数据字段。
- 修正删除活动前缺少二次确认的风险。
- 增加表单错误反馈，而不只依赖按钮禁用。
