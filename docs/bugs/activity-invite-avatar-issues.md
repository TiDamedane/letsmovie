# 活动邀请与头像测试问题

## 背景

来自当前活动详情页邀请/头像流程测试。

## 状态

全部记录项已处理：上传头像重试、电脑端上传失败 fallback、邀请弹窗自动出现、主持人头像来源、成员区邀请入口与成员头像排版、头像裁剪交互均已修复。

## 修复记录

- 2026-06-21：修复问题 1 和问题 6。云端头像上传失败时不再阻断加入流程，会回退为本地 data URL；重复点击上传头像时会清空 file input，允许重新选择同一张图片。
- 2026-06-21：修复邀请弹窗相关问题 2、3、4、7、8。随机创建活动后的邀请弹窗触发保留；主持人头像改为读取真实 host participant；成员区只展示真实 member，邀请加号保持为右侧入口；成员头像以 `40px x 40px` 圆形、`12px` 间距排在邀请按钮左侧，新成员继续往左扩展。
- 2026-06-21：修复问题 5。头像裁剪从三条 range 控制改为圆形预览内直接拖动、滚轮缩放、双指捏合缩放；重新选择头像改为预览右下角加号入口。

## 问题清单

1. 头像上传失败后应允许用户反复重新选择/上传，不能第一次失败后入口失效。
2. 随机创建观影局后进入活动详情页，应自动弹出“邀请好友”弹窗。
3. 被邀请用户之后创建自己的活动时，主持人头像应使用该用户加入时选择的头像，而不是固定默认猫头像。
4. 多成员加入时，成员头像区需要支持多人排版：头像 `40px x 40px`，圆形，从右往左排列，间距 `12px`。
5. 头像裁剪应改成社交软件常见的手势缩放/拖动裁剪，不要三条 range 控制条。
6. 需要排查电脑端上传头像失败的原因。
7. 主持人点击成员区邀请按钮后，无论复制邀请还是稍后再说，弹窗消失后不应把邀请加号替换成主持人自己的头像；成员区只能展示真实 member，不能展示 host/current user 的头像。
8. 邀请按钮在随机模式执行“开始选片”前、确认模式执行“留下回忆”确认前都不能消失；有新成员加入时，成员头像按从右往左规则排列在邀请按钮左边，头像大小不变、间距 `12px`。

## 原实现线索

- 主持人头像目前仍有写死默认图的路径。
- 成员栏目前只展示第一个 member。
- 成员列表来源需要排查是否把当前 participant、remote participants 或 memories 中的主持人误归类为 member。
- 原头像裁剪 UI 使用 range 控制缩放/x/y。
- Supabase Storage 上传失败会导致头像上传报错。

## 后续修复验收

- 上传失败后可重新选择头像并再次提交。
- 创建随机观影局后邀请弹窗只自动出现一次。
- 新创建活动的 host 头像来自当前本地 participant。
- 邀请按钮在活动进入挑选/回忆确认前始终保留为右侧入口。
- 多成员头像在邀请按钮左侧从右往左排列，`40px x 40px` 圆形，间距 `12px`。
- 邀请弹窗关闭后，若没有真实成员，成员区仍显示 `40px x 40px` 圆形邀请加号。
- 裁剪交互支持拖动和缩放。
- 电脑端上传失败原因被定位并修复或给出明确 fallback。
